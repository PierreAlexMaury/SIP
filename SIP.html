SIP

Context

SIP is now the most used protocol for Internet telephony or Voice over IP (VoIP). It also can be used for instant messaging, virtual reality, video teleconferencing, or video games. These multimedia content is exchanged between session participants using an appropriate transport protocol. In many cases, the transport protocol to use is the Real-Time Transport Protocol (RTP).

SIP has gradually replaced the H.323 protocol, which is used for call signaling and control, multimedia transport and control, and bandwidth control for point-to-point and multipoint conferences.

It is now the industry standard for the telecom and multimedia communications. Half of the total number of users of a telecom service has used or will use at least once SIP-based services. So the spending to build network infrastructure could reach 10 billion euros.

The main advantage of the opening of a free SIP account or SIP trunking with a VoIP provider is the fact that it is possible to make free calls to all other users on other SIP providers in the world by simply composing their SIP addresses, provided that the SIP network operator is opened.

The only major player not using the SIP protocol is the "Skype" service, which deprives him of being compatible with a lot of hardware, software and market SIP applications.

Terminology

A system using SIP can be viewed as consisting of components defined on two dimensions [6]: client/server and individual network elements. Client and server are defined in the RFC 3261 by:

• Client: A client is any network element that sends SIP requests and receives SIP responses. Clients may or may not interact directly with a human user. User agent clients and proxies are clients.

Then in a standard SIP configuration we find individual elements, in the category of client we have the Agent Equipement (AE) which represents:

• User Agent: The user agent resides in every SIP end station[6]. It is a SIP network terminal (SIP telephones, or gateway to other networks), it can be:


» User Agent Client (UAC): Client in the terminal that initiates SIP signalling

» User Agent Server (UAS): Receives SIP requests and generates a response that accepts, rejects, or redirects the request.


In SIP, as in HTTP, the user agent may identify itself using a message header field 'User-Agent', containing a text description of the software/hardware/product involved. The User-Agent field is sent in request messages, which means that the receiving SIP server can see this information. We can also note that a single user agent may function as an UAC and an UAS.



• Server: A server is a network element that receives requests in order to service them and sends back responses to those requests. Examples of servers are proxies, user agent servers, redirect servers, and registrars.

Proxy server (PS): It is an intermediary entity that acts as both a server and a client for the purpose of making requests on behalf of other clients.
A proxy server primarily plays the role of routing, meaning that its job is to ensure that a request which is received for a connexion is sent to another entity closer to the targeted user if this particular station is not in its administration.
This request is sent by the client because the exact address of the recipient is not known in advance. Then this request can be forward to another proxy server or the recipient itself.

Redirect Server (RS): It is used during session initiation to determine the address of the called device. This server accepts SIP requests, translates the address contained in the SIP request in zero or more network addresses if it has been recognized or not and finally returns them to the client to contact an alternate Universal Resource Identifier (URI) which is a generic identifier used to name any resource on the Internet.

Location Server (LS): It is used by a SIP redirect or proxy server to obtain information about a callee's possible location(s) after have received registration requests from the UA. This function is provided by the location service composed of a database or a text file containing the list of UA, their rights and their password.

Registrar (RG): To detect the location of an user in a network (it is one of a prime works of a server) the user have to register his location to a Registrar server and refreshes from time to time his location by sending a REGISTER request to a Registrar server.
So a Registrar server is a SIP endpoint that accepts REGISTER requests and places the information it receives in those requests into a location service for the domain it handles. The location service links one or more IP addresses to the SIP URI of the registering agent. The URI uses the SIP scheme, although other protocol schemes are possible. More than one user agent can register at the same URI, with the result that all registered user agents receive the calls to the URI.

All these server sections (Location,Proxy,Redirect...) are often on a single physical machine: a proxy server, which is responsible for client database maintenance, connection establishing, maintenance and termination, and call directing.
