Introduction

The Session Initiation Protocol, defined in RFC 3261 "SIP: Session Initiation Protocol", is an application level signaling protocol for setting up, modifying, and terminating real-time sessions between participants over an IP data network. SIP is originally defined by Multiparty Multimedia Session Control (MMusic) and is now normalized and standardized by the Internet Engineering Task Force (IETF). This protocol is based on a transactional client/server model which is similar to Hyper Text Transport Protocol (used to surf the web) and Simple Mail Transport Protocol (used to transmit electronic messages). SIP can be used for two-party (unicast) or multiparty (multicast) sessions and it incorporates the use of a Session Description Protocol (SDP), which defines session content using a set of types similar to those used in Multipurpose Internet Mail Extensions (MIME).
